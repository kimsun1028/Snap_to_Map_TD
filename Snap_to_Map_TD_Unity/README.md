# Snap_to_Map_TD_Unity

Unity side of the Snap to Map TD workflow.

## What this project does
- Imports `path_data.json` generated by `Mapmaking.py`
- Loads the path from `Assets/StreamingAssets/path_data.json`
- Converts pixel coordinates into Unity world coordinates
- Moves enemies along the generated tower-defense route

## Setup
1. Open `Snap_to_Map_TD_Unity` in Unity 6.3.
2. In the editor, run `Tools > Snap To Map > Import Map Assets from Repo Root`.
3. Add `MapManager` to a scene object.
4. Add `PathFollower` to your enemy prefab.
5. If you want to visualize the route, add `PathGizmoDrawer` to any scene object.

## In-game photo preview
- `MapManager` now loads `img.jpg` from `Assets/StreamingAssets/img.jpg`.
- The route is drawn on top of the photo so you can check whether the edge tracking matches the image outline.
- The imported photo is fixed to the repo-root `img.jpg`.

## Coordinate mapping
- `path_data.json` stores pixel coordinates from the source image.
- If `image_size` exists in the JSON, the loader uses it automatically.
- If `image_size` is missing in an older JSON file, set `mapPixelSize` manually once.
- `worldSize` controls how large the path is in the scene.
- `plane` can be `XZ` for 3D top-down or `XY` for 2D.

## Menu actions
- `Tools/Snap To Map/Import Path JSON from Repo Root`
- `Tools/Snap To Map/Open StreamingAssets Folder`
